## Pitch Deck Prompts (6)

### 1. Pitch Deck Story Arc Creator
```
Help me craft the narrative arc for my startup pitch deck.

Startup: [what you're building]
Problem: [the pain point you solve]
Solution: [how you solve it]
Target customer: [who you serve]
Traction so far: [users, revenue, milestones]
Ask: [what you're raising/seeking]

Create a compelling story arc that:
1. Opens with a hook that captures attention
2. Builds emotional connection to the problem
3. Positions my solution as the inevitable answer
4. Demonstrates momentum and credibility
5. Ends with a clear, compelling ask

Provide the narrative flow and key talking points for each of the 10-12 slides.
```

### 2. Problem Slide Optimizer
```
Help me make my problem slide compelling and memorable.

The problem I'm solving: [describe in detail]
Who has this problem: [target customer]
How they currently solve it: [existing solutions]
Why current solutions fail: [gaps and frustrations]
Cost of the problem: [money, time, pain]

Create 3 versions of my problem slide:
1. Data-driven version (stats and numbers)
2. Story-driven version (customer narrative)
3. Visual concept version (analogy or metaphor)

For each, provide:
- Headline
- 3 supporting points
- Memorable stat or quote
- Suggested visual/image concept
```

### 3. Traction Slide Maximizer
```
Help me present my traction in the most impressive way.

Our metrics:
- Users/customers: [number]
- Revenue: [amount and timeframe]
- Growth rate: [percentage]
- Other metrics: [engagement, retention, etc.]
- Timeline: [when you started]
- Key milestones: [achievements]

Even if numbers are small, help me:
1. Frame metrics in the best light
2. Show growth trajectory
3. Highlight leading indicators
4. Compare to relevant benchmarks
5. Tell the story of momentum

Create multiple ways to present this traction, including:
- The key headline metric
- Supporting proof points
- Graph/chart suggestions
- Before/after comparison
```

### 4. Competitive Landscape Slide
```
Create a compelling competitive landscape slide.

My startup: [what you do]
Direct competitors: [list them]
Indirect competitors: [alternatives]
My key differentiators: [what makes you unique]

Create:
1. 2x2 matrix positioning (with axis labels that favor me)
2. Feature comparison table (choosing features I win on)
3. "Why we win" narrative
4. Counter-arguments for "why not just use X"
5. Market gap visualization concept

Make sure my positioning feels inevitable, not defensive.
```

### 5. Team Slide Storyteller
```
Help me present my founding team compellingly.

Team members:
[Name 1]: [background and role]
[Name 2]: [background and role]
[Add more as needed]

Our collective experience includes: [relevant highlights]
Why we're uniquely positioned: [unfair advantages]
What's missing: [roles we need to hire]

Create:
1. Team narrative (why this team for this problem)
2. Bios (2-3 sentences each, focused on relevance)
3. Social proof highlights
4. "Unfair advantage" framing
5. How to address team gaps honestly
```

### 6. Ask Slide & Close
```
Help me create a compelling ask and closing slide.

Raising: [amount]
Valuation/terms: [if known]
Use of funds: [how you'll spend it]
Milestones to hit: [what this gets you to]
Timeline: [runway and goals]

### 8. Cold Email to Investors
```

Traction: [key metrics]
Team: [brief background]
Raising: [amount]
Why now: [why this is timely]

Create 3 email versions:
1. Ultra-short (under 100 words)
2. Standard (150-200 words)
3. Story-driven (with a hook)

For each:
- Subject line options (5)
- Opening hook
- Value proposition
- Social proof element
- Clear ask
- Follow-up sequence (3 emails)

Avoid cliches like "the Uber of" or "revolutionary."
```

## MVP & Product Prompts (6)

### 14. MVP Scope Definer
```

Problem I'm solving: [the pain point]
Full vision: [what I want to build eventually]
Target user: [who uses this first]
Core value proposition: [why they'd use it]
Resources: [time, money, team]
Timeline: [when I need to launch]

Help me:
1. Identify the ONE core job-to-be-done
2. List features (must-have vs. nice-to-have)
3. Cut scope ruthlessly (what can wait)
4. Define "done" for MVP
5. Identify riskiest assumptions
6. Create validation criteria
7. Set success metrics for MVP
8. Plan post-MVP learning roadmap
```

### 15. User Story Generator
```
Generate user stories for my MVP.

Product: [what you're building]
Core users: [who uses it]
Main use cases: [what they do with it]

1. Primary user story
2. Acceptance criteria
3. Edge cases to consider
4. Dependencies

6. Estimated complexity (S/M/L)

Group by epic/theme and prioritize for MVP.

5. **Take action** - Implement what you learn
